01 WHAT IS A DIGITAL TWIN? DEFINITION & HISTORY

⏸ SIMULATION

  • Static snapshot in time
  • Predefined scenarios only
  • Shows how things should work
  • Run once — then it's done
VS

▶ DIGITAL TWIN

  • Continuously updated via live sensors
  • Mirrors real-world conditions right now
  • Shows how things are working
  • Learns over time via machine learning

📡THE DEFINITION

"A virtual representation of a real-world object, system, or process — where sensors continuously feed live data into the digital model, ensuring it reflects real-world conditions in real time." — NSF, 2026

🔄THE KEY DIFFERENCE

A digital twin isn't just a 3D model — it's a living system that breathes with its physical counterpart. Every sensor reading, every temperature shift, every performance metric flows into the twin in real time.

02A THE SENSOR LAYER — HOW TWINS FEEL THE WORLD SOURCE: DATAKNOW · IBM · NASA

Sensors are the nervous system of a Digital Twin — without them, the virtual model is just a 3D picture. Sensors capture the physical world's real-time state and transmit it continuously so the twin stays alive and accurate.

ENVIRONMENTAL SENSORS
TEMPERATURE
Thermocouples, infrared sensors — engine heat, component stress
PRESSURE
Transducers, gauges — hydraulics, fuel lines, cabin pressure
HUMIDITY
Capacitive sensors — corrosion risk, avionics protection
LIGHT / EM
Photodiodes, ambient sensors — targeting optics, cockpit display
MOTION & POSITION SENSORS
POSITION
Linear encoders, rotary sensors — control surface angles, landing gear
ACCELEROMETERS
IMUs, gyroscopes — G-force, roll/pitch/yaw, vibration detection
GEOSPATIAL / GPS
GPS, compass, altitude sensors — precise navigation and terrain mapping
METEOROLOGICAL
Anemometers, wind sensors — live weather modeling, flight path decisions
DETECTION & IMAGING SENSORS
IMAGE / CAMERA
CCD, CMOS, thermal imaging — target ID, damage assessment, ISR feeds
LIDAR / 3D RADAR
Ultrasonic, LiDAR, 3D cameras — real-time obstacle mapping, terrain modeling
ACOUSTIC / SOUND
Microphones, piezoelectric — engine signature, structural crack detection
FORCE / STRAIN
Load cells, strain gauges — structural integrity, weapons rack stress
SPECIALIZED SENSORS
CHEMICAL / GAS

BIOMETRIC

VIRTUAL SENSORS

ELECTROMAGNETIC

02 THE 9 TYPES OF DIGITAL TWINS IBM MODEL FRAMEWORK
Digital twins exist on a spectrum — from a single bolt on a jet engine all the way up to an entire city. IBM defines 9 types across two dimensions: scope (component → process) and intelligence (descriptive → autonomous).
TYPE 01
Component Twin
Tracks one specific part — like a single turbine blade. Most granular level of analysis.
TYPE 02
Asset Twin
Multiple components working together — studies interactions and generates actionable performance data.
TYPE 03
System Twin
Broader view showing how multiple assets combine into one fully functioning system.
TYPE 04
Process Twin
Reveals how all systems work across an entire production facility — identifies bottlenecks and inefficiencies.
TYPE 05
Descriptive
Live-editable visual replica of assets or facilities. Design and construction data in real time.
TYPE 06
Informative
Adds a layer of operational and sensor data. The more data added, the stronger the link to its physical twin.
TYPE 07
Predictive
Leverages operational data to forecast future behavior — anticipates failures before they happen.
TYPE 08
Comprehensive
Simulates possible future scenarios — asks "what if?" questions and models multiple outcomes.
TYPE 09
Autonomous
The frontier — learns and acts on behalf of users. Closes the loop between insight and physical action.

ALTITUDE SENSORS
Radar Altimeter + Pitot-Static System. Radar bounces signals off terrain for AGL readings; pitot-static measures air pressure differentials for MSL altitude. GPS/GNSS provides position confirmation. Together they give redundant altitude truth to the twin.
PAYLOAD SENSORS
Strain Gauges + Load Cells on wing pylons. Each hardpoint has embedded load cells that measure weight distribution and stress continuously. The twin uses this data to recalculate center-of-gravity, flight envelope limits, and G-tolerance in real time.
FUEL SENSORS
Capacitance Fuel Level Sensors + Flow Meters. Capacitance sensors measure dielectric change as fuel level drops in each tank. Flow meters at engine inlet track real consumption rate. The twin calculates burn rate, remaining endurance, and safe return thresholds dynamically.
HARDPOINT / WEAPONS SENSORS
Electrical Continuity + Armament Bus Sensors. Each pylon has continuity sensors confirming weapon attachment, armed status, and release circuit integrity. The armament management system feeds all 8 station states to the twin, which models aerodynamic drag and weight change after each release event.
STEALTH / RCS SENSORS
RF Emission Monitors + Radar Warning Receiver (RWR). The RWR detects incoming radar frequencies and signal strength. RAM (Radar Absorbing Material) coating sensors detect surface damage that degrades stealth. The twin models the drone's current radar cross-section based on known geometry and coating integrity in real time.
